DATA MANAGEMENT

数値が合わない時は、手順通りに。

1. 全体像と使うツール

このマニュアルは、POTEXの顧客管理・面談管理・着金管理・数値管理を迷わず運用するためのものです。

まず覚えること3つ
1. Slackで報告された内容は、必ず該当シートに反映する。
2. 面談結果は、面談管理シートとLSTEPの両方を更新する。
3. 数値が合わない時は、3章の確認順に沿って、LSTEPタグ・CSV更新・Slack報告未反映を先に確認する。
基本の流れ
LSTEP予約・タグ / Slack報告 → 面談管理・着金記録・受講者管理 → 数値管理シート
ツール / シート主な用途注意点
LSTEP LINE顧客情報、予約管理、タグ管理、顧客ID確認 タグ漏れがあると数値が合わなくなる
Slack CS・営業・コーチ・運営間の報告、リマインド、着金報告 報告内容をシートに反映する必要がある
TIMEREX 一部面談の日程調整 TIMEREXで調整された面談はSlack報告とあわせて確認する
面談管理シート 無料体験・再面談・営業結果の管理 Slack報告とLSTEP更新の両方を確認する
着金記録 支払額・着金履歴の管理 着金履歴を残す
受講者管理 成約後の受講者情報、マッチング、AFS管理 入力漏れがよく発生
数値管理シート 経営・運営確認用の集計 元データの入力漏れに注意する
数値管理シートや受講者管理などの自動計算セルを編集する場合は、参照範囲・集計条件を確認してから修正する。

2. 各業務の手順

2-1. 【予約時 / CS・営業】初回無料体験日程調整

  1. LSTEP予約管理で顧客名、予約日時、担当営業を確認する。
  2. LSTEPタグを追加する。
  3. TIMEREXで日程調整された一部面談がある場合は、Slack報告とあわせて確認する。

2-2. 【日中 / CS・運営】無料体験前日のリマインド

  1. [CS] 日中に翌日の無料体験予約を確認し、顧客にLINE（LSTEP）でリマインドを送信する。
  2. Slackの「営業_盛り上げグループ」に営業メンバー向け周知を投稿する。
  3. 面談管理シートに、顧客名・予約日時・担当営業・流入経路を記入する。

2-3. 【面談後 / 営業・運営】無料体験後の営業報告

  1. Slackの該当リマインド投稿へのコメントを確認する。
  2. 面談結果、担当者、面談日、ステータスを面談管理シートに反映する。
  3. LSTEPのステータスタグ（成約／失注／再面談など）を追加する。
  4. 友だち情報の「成約日」「失注日」など、必要な日付項目を入力する。
  5. 成約時は、プランタグが追加されているか確認し、なければ追加する。

2-4. 【着金後 / 運営】着金確認

  1. Slackの着金報告を確認する。
  2. 着金記録に顧客ID、顧客名、今回の着金額、支払い方法、備考を入力する。
  3. 顧客IDはシートに検索用の関数があるため、本名から検索する。
  4. 分割支払いの場合は既存行を上書きせず、新しい1行として追加する。

2-5. 【着金後〜面談前 / CS・コンシェルジュ・運営】成約後の受講者管理

  1. 着金確認後、原則コンシェルジュ面談前に受講者管理シートを更新する。
  2. マッチング連絡日、顧客ドライブURL、プラン、担当者を入力する。
  3. 最終セッション完了見込は、マッチング連絡日とプランをもとに自動計算される。

2-6. 【随時 / コーチ・AFS担当】AFS管理

  1. Slackの「afs_マッチング部屋」を確認する。
  2. AFS予定日とAFS結果を受講者管理に入力する。
  3. Slack上で報告されていても、受講者管理に未反映の場合があるため照合する。

2-7. 【随時 / コーチ・運営】継続プラン成約

  1. 継続プランが成約した場合は「継続プラン」タブに記入する。
  2. 成約日、顧客名、プラン内容、支払額、着金状況、担当者を確認する。

2-8. 【数値確認前 / 運営】LSTEP CSV更新

  1. LSTEPを開く。
  2. 「友だちリスト」へ移動する。
  3. CSVエクスポートリストを開く。
  4. 「Potential X」と「RYU個別コーチング」をダウンロードする。
  5. 数値管理シートを開く。
  6. 「Potential X csv」「RYU個別コーチング csv」タブの既存貼付範囲にCSVをインポートする。
  7. 数値管理シートの集計結果を確認する。
CSV更新を忘れると、LSTEP上では最新でも数値管理シートには反映されない。数値確認前には必ずCSVが最新か確認する。

3. 困った時の確認順

3-1. 数値が合わない時

  1. LSTEPの完了タグ・成約/失注/再面談タグが正しいか確認する。
  2. 流入経路タグを確認する。リファラル経由の場合は4-1を参照する。
  3. Potential X / RYU個別コーチングのCSVが最新か確認する。
  4. 面談管理シートのステータスが空白・報告なし・検討中のまま残っていないか確認する。
  5. Slack報告が面談管理・着金記録・受講者管理に反映されているか確認する。

3-2. 起こりやすいミスと対処法

ミス原因対処法
成約数が少ないLSTEP成約日タグ漏れLSTEPタグを確認して追加
面談数が合わない面談管理シート未入力Slack報告を確認して反映
売上が合わない着金記録漏れSlack着金報告と照合
同じ顧客が重複している両方の公式LINEに登録本名・メールアドレス・予約日時で照合
分割支払いが反映されない既存行を上書きした分割分を新しい行で追加
AFS対応漏れSlack報告のみでシート未反映afs_マッチング部屋と受講者管理を照合
ステータスが古い営業報告後の更新漏れ面談管理とLSTEPを更新

5. 引き継ぎチェックリスト

5-1. 権限・環境
  • ☐ LSTEPのログイン・閲覧権限がある
  • ☐ Slackの必要チャンネルに入っている
    （営業_盛り上げグループ、afs_マッチング部屋 など）
  • ☐ 面談管理シートを編集できる
  • ☐ 着金記録を編集できる
  • ☐ 受講者管理を編集できる
  • ☐ 数値管理シートを編集できる
5-2. 操作理解
  • ☐ LSTEPタグの追加方法を理解している
  • ☐ 顧客の検索方法を理解している
  • ☐ Potential X / RYU個別コーチングCSVの更新方法を理解している
  • ☐ 数値が合わない時の確認順を理解している
  • ☐ 自動計算セルを直接編集する場合は、参照範囲・集計条件を確認することを理解している
  • ☐ CSV貼付開始セル・範囲を実画面で確認した
5-3. 日次チェック
  • ☐ 翌日無料体験を確認した
  • ☐ 顧客へのリマインドを送信した
  • ☐ Slackに営業メンバー向け周知を投稿した
  • ☐ 営業報告を面談管理シートとLSTEPに反映した
  • ☐ 着金報告を着金記録に反映した
5-4. 随時チェック
  • ☐ 成約者を受講者管理・LSTEPタグに反映した
  • ☐ AFS予定・結果を確認した
  • ☐ 継続プラン成約時に該当タブへ記入した
  • ☐ 数値確認前にCSVが最新か確認した
